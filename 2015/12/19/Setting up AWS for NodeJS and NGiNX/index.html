Setting up AWS for NodeJS and NGiNX

Configure SSH, login as ec2-user

Install prerequisites.

Install NodeJS from Binaries

Update NPM to Latest

Format presented volume to ec2 instance.

From the docs http://docs.aws.amazon.com/AWSEC2/latest/UserGuide/ec2-add-volume-to-instance.html

Use lsblk to determine the volume that was presented.

Mount the device.

Create a new user on AWS EC2 Linux image.

http://docs.aws.amazon.com/AWSEC2/latest/UserGuide/managing-users.html

Create a new keypair in the AWS console.
http://docs.aws.amazon.com/AWSEC2/latest/UserGuide/ec2-key-pairs.html#having-ec2-create-your-key-pair

After copying and securing the .pem file you need to add the private key to the ssh keystore.

Next, export the public key so you can install that on the server. (Perform this command on your workstation)

  • You will be asked for the path to your private key.

The public key will dump to the screen, copy this entire string: (This is done on your workstation)

Now, login to your AWS EC2 server instance and edit the authorized_keys file you created earlier.

Paste the ssh-rsa string into this file and save it.

You can now login to EC2 from your workstation.
